## 3. Takeaway 提炼（从 Karpathy gist）

1. Persistent wiki vs RAG：wiki 是 compounding artifact，RAG 是每次现场重拼
2. 三层架构：raw (immutable) / wiki (LLM-written) / schema
3. 三操作 + File Back 从属：File Back 原本在 Query 下，不是独立动作（我们项目把它独立化——**显式化而非违反**）
4. log.md 格式原文就是 `## [YYYY-MM-DD] ingest | Title`——我们**严格匹配原始约定**
5. 核心比喻："Obsidian is the IDE; the LLM is the programmer; the wiki is the codebase"
6. 为什么可行：bookkeeping 是知识库真正瓶颈，LLM 不 bored
7. 历史继承 Vannevar Bush Memex (1945)：Bush 的愿景 > Web，LLM 解维护难题

## 5. Gap-2 决策的决定性证据（来自本次 ingest）

**来自 raw 原文**（§Architecture / Raw sources）：

> "These are immutable — the LLM **reads from them but never modifies them**."

**关键词**："never modifies"，不是 "never writes"。规则核心是**内容完整性**，不是**物理写入权限**。

**旁证**（§Tips）：
> "**Obsidian Web Clipper** is a browser extension that converts web articles to markdown. Very useful for quickly getting sources into your raw collection."

Karpathy 主动推荐一个 browser extension 往 raw/ 写内容——说明**自动化工具 write to raw/ 是设计意图的一部分**。

**结论**：我们原 CLAUDE.md 写 "AI 绝不能 Write / Edit raw/"**比 Karpathy 原意严一档**。Gap-2 执行（松绑至"禁 Edit 已存在文件 + 授权下可 Write 新文件 with provenance"）是**对原意的纠正**，不是偏离。
